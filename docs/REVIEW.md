# REVIEW

## 功能

1. `实时档同步` card 中的功能应该提取出来到 header 中，因为此功能不应该放到配置界面中，它是一个全局的功能

2. `备份` card 应该是作为一个单独的弹窗存在，它应该具备管理功能，具体操作应该有：恢复、删除、导出，备份

3. `存档状态` 不要以json形式展示，应该以 label - value 形式展示，并且有些可更改项应该可以进行编辑，比如 `存档路径`， `实时路径`， 等。

4. `日志` 要放到一个 card 里，并且每一条日志前面要加上事件发生时间

5. 每一个 card 中如果进行编辑，则该card 的label上应该标注一个星号

## UI

1. 界面布局使用 grid 布局，要把横向空间占满，每个 card 有独立的y方向 scroll，x方向的宽度保持最小值，最大值不限，保持 grid 的响应性。

2. `应用到缓冲` 这种按钮，应该是放在 card label 平行的最右侧，与其接近的说明，应该用最小的字体放到 label 的下方。

3. `法杖与法术` card 应该统一使用一个 `应用到缓冲` 按钮，而不是每个法杖一个，这会导致一个法杖保存时，其他法杖未被保存的数据被重置，背包法术(inventory_full) 这一项提到card的最上面

4. header 的层级没有 info 图标高，以至于 info 图标都在 header 之上

## i18n

1. 界面至少应该包含中文和英文两种语言，并且可以随意切换

## 工程相关

1. 是否将代码从 editor 中拿出来更好？

2. 后期准备将其打包成一个跨平台的可执行文件，预计用 Tauri 或者 Electron 来实现，请做技术选型。
